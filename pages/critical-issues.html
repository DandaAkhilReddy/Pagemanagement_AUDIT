Issue #1: Duplicate Encounter Problem

Impact Assessment

  • Staff Efficiency Loss: 40% rework rate
  • Accounts Affected: 2-3 touches per duplicate
  • Facilities Most Impacted: Palms Hospital, JFK Medical Center
  • Time Wasted: ~15 hours/week on duplicate management

Root Cause Analysis

Primary Cause: 24-Hour System Lag

The Problem:

  • Patient seen on Day T doesn't appear in system until Day T+1
  • Providers can't wait 24 hours to document charges
  • Providers manually create patient records to bill same day
  • When census team enters patient next day, duplicate is created

System Architecture Issue:

  • IngeniousMed → AdvancedMD sync runs once daily
  • No real-time API available
  • No "pending sync" indicator for providers
  • No duplicate detection before creation
Secondary Cause: Census Processing

Census Duplication Issues:

  • Adjacent-day census PDFs contain same patients
  • Example: Patient on 11/16 census also appears on 11/17
  • No automated comparison between days
  • Manual review often misses duplicates

Proposed Solutions

Immediate Mitigation (Week 1)

  • Sort census by Name + DOB before entry
  • Compare adjacent-day census lists side-by-side
  • Flag potential duplicates: Same DOS + LOS + Provider
  • Verify admit/discharge dates match

Short-term Fix (Weeks 2-4)

  • Build duplicate detection report in Excel/SQL
  • Match on: Name, DOB, DOS, Provider, LOS
  • Run daily before billing submission
  • Export "Review Duplicates" CSV for staff

Long-term Solution (Months 2-3)

  • Implement real-time IM ↔ AMD sync
  • Add "pending sync" indicator in IngeniousMed
  • Pre-bill duplicate validator in AMD
  • Automated merge functionality
