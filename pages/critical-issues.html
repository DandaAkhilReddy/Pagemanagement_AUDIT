Issue #1: Duplicate Encounter Problem

Impact Assessment

  • Staff Efficiency Loss: 40% rework rate
  • Accounts Affected: 2-3 touches per duplicate
  • Facilities Most Impacted: Palms Hospital, JFK Medical Center
  • Time Wasted: ~15 hours/week on duplicate management

Root Cause Analysis

Primary Cause: User Error in Patient Search

The Problem:

  • Doctors cannot quickly locate existing patients in IngeniousMed search
  • Under time pressure, doctors create new patient records instead of thorough searching
  • This creates duplicate patient entries in the system
  • Duplicate records lead to 2-3x rework on billing and reconciliation

Why This Happens:

  • Patient search requires exact matching or careful filtering
  • High-volume environment with time constraints
  • Doctors prioritize patient care over system navigation
  • No clear warning that patient may already exist
Secondary Issue: IngeniousMed Assignment Lag

IngeniousMed System Update Impact:

  • IngeniousMed system update caused 1-day assignment delay
  • Example: On November 14th, can only assign encounters from November 13th
  • If doctor bills same day: Charges appear immediately and can be processed
  • If doctor doesn't bill: Reshma's team cannot assign encounters until next day

Workflow Impact:

  • This is an operational workflow issue, NOT a cause of duplicates
  • Today's billing only populates if doctor adds charges same day
  • Otherwise, assignment team works on yesterday's encounters

Proposed Solutions

Immediate Mitigation (Week 1) - User Training

  • Train doctors on effective patient search techniques in IngeniousMed
  • Create quick reference guide for patient lookup best practices
  • Emphasize importance of thorough search before creating new patients
  • Implement "double-check" workflow: search twice before creating new record

Short-term Fix (Weeks 2-4) - System Improvements

  • Improve IngeniousMed search functionality (fuzzy matching, suggestions)
  • Add warning message: "Similar patient found - check before creating new"
  • Build duplicate detection report in Excel/SQL for billing team
  • Run daily duplicate check before claim submission

Long-term Solution (Months 2-3) - Automation

  • Implement AI-powered duplicate detection before patient creation
  • Add real-time patient matching suggestions during search
  • Create automated merge functionality for confirmed duplicates
  • Dashboard showing duplicate creation trends by provider
