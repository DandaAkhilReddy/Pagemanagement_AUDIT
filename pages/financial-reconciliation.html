AR Movement Methodology

Dollar-Based Tracking (Preferred)

  • Shows actual cash movement
  • Not skewed by charge volume changes
  • Clear improvement/deterioration visibility
  • Direct correlation to cash flow

Percentage-Based Issues

  • Big charge spikes distort percentages
  • Current bucket inflates when charges increase
  • Misleading when comparing periods
  • Use only as secondary metric

Reconciliation Workflow

Current Manual Process

Step 1: Weekly Bank Register
  • Chris exports bank deposits
  • Sends Excel file to Lisa
  • Shows date, amount, description
  • Includes check/EFT numbers
Step 2: EOB Compilation
  • Lisa compiles week's EOBs
  • Lists by payer and amount
  • Notes expected deposit dates
  • Tracks batch numbers
Step 3: Manual Matching
  • Match amounts first
  • Use date ranges based on payer lag
  • Identify unmatched deposits
  • Flag missing EOBs
Step 4: Investigation
  • Research unmatched items
  • Contact payers for missing EOBs
  • Check for split deposits
  • Review posting errors

Proposed Automated Process

Automation Components:

  • Bank API integration: Auto-import daily deposits
  • ERA auto-matching: Match by amount + date range
  • Payer mapping table: Handle name variations
  • Exception reporting: Flag unmatched items daily
  • Variance dashboard: Real-time discrepancy view
