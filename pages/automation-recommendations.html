Phase 1: Quick Wins (0-4 Weeks)

Duplicate Detection System

Implementation Details:

  • Technology: SQL query or Python script
  • Data Points: Name + DOB + DOS + Provider + LOS
  • Process: Daily run before billing submission
  • Output: "Review Duplicates" CSV file
  • Time to Build: 1 week
  • Testing: 1 week

Payer Deposit Lag Monitor

Automated Tracking System:

  • Track EOB date vs actual deposit date
  • Calculate average lag by payer
  • Flag deposits exceeding expected lag
  • Auto-email payer with claim references
  • Dashboard view of all pending deposits

SFTP Watchdog

File Upload Detection
  • Monitor SFTP folders for new files
  • Log timestamp of upload
  • Start 24-hour countdown timer
Pull Verification
  • Check if file accessed/downloaded
  • If not pulled in 24 hours, send notification
  • Escalate after 48 hours
Auto-Archive
  • Move pulled files to archive folder
  • Maintain 90-day archive
  • Auto-delete after retention period
